学习一下Git的常用命令, 以便后面的学习与工作中使用Git

Git常用命令

这两天在学习Git的使用, 虽然之前也使用过, 但仅限于add commit push pull等命令, 对于分支什么的都没用过, 所以找了教程来看看, 主要参考廖雪峰老师的教程和一个在线Git练习网站

注: 不会将参考教程时实际演示的指令记录下来, 因为这是不必要的

让我们开始吧

配置Git

安装Git后执行以下命令

git config --global user.name "Your name"
git config --global user.email "Your email"

Git是团队使用的版本控制系统, 每一次提交都是需要自报家门的, --global参数是表示该机器上所有的Git仓库都使用该配置, 当然也可以分不同的配置, 不过就不提了, 这个配置很适合个人机器

配置ssh秘钥

ssh-keygen -t rsa -C "Your email"

创建版本库

git init # 创建一个版本库

状态查看

git status # 查看仓库的状态, 是否有修改, 是否add

添加到暂存区

git add <file name> # 将工作区修改的改动添加到暂存区, 支持文件夹, 例如git add . 添加整个当前目录下所有的改动

提交到仓库

git commit -m "message" # 对当前commit做一个简短的说明

查看区别

git diff <file name> # 前提是未提交到仓库的修改, 可以查看当前修改后的文件和最新的仓库版本中的同文件的改动

提交日志

git log # 查看所有的commit即提交说明
git log --graph # 查看分支合并图

版本回退

git reset [--soft | --mixed | --hard] <commit> # 回退到某个版本, --hard参数是不保存已存在的修改,　其他两个参数是保存修改, 默认参数是--soft

历史命令

git reflog # 显示你使用过的历史命令

撤销修改

git checkout -- <file name> # 若文件在暂存区, 撤销修改就和版本库一样, 若文件在暂存区了, 撤销修改就回到了添加到暂存区后的状态
git reset HEAD <file name> # 将暂存区的文件退回到工作区
# 场景1: 文件还没有添加到暂存区, 使用git checkout -- <file name> 丢弃文件
# 场景2: 文件已经添加到暂存区, 使用git reset HEAD <file name> 回到了场景1, 若要丢弃文件, 执行场景1, 若修改, 正常来即可, reset可以将暂存区的文件保存原理看[版本回退]

删除被Git追踪的文件

# 先要删除文件
git rm <file name> # 从版本库中删除该文件
# 另一种是删错了
git checkout -- <file name> 使用版本库中的版本替换工作区中的版本

远程仓库

git remote add origin git地址 # 将本地仓库与之关联, 这是先有本地仓库, origin就是远程仓库的名字, 也可以使用其他的, 但不推荐, 
git clone git地址 # 这是先有远程仓库, 将其克隆下来

push

git push -u origin master # 第一次提交, 加上-u参数, 为以后提交简化命令, 默认参数就是origin master, 后面可以不用写, 但是你的远程还有一个dev分支的话, 要提交dev上就要指定参数了

创建分支

git branch <分支名> # 创建分支
git checkout <分支名> # 切换分支
# 可以使用git checkout -b <分支名> 同时执行上面两条指令

git switch <分支名> # 切换分支, 但是是新版Git才有的功能
git switch -c <分支名> # 创建并切换分支

git checkout -b <本地分支> <origin/远程分支>, 创建远程分支并关联到本地分支
git branch --set-upstream-to=<origin/远程分支> <本地分支> # 设置本地分支和远程分支的关联

查看分支

git branch # 查看当前仓库的分支

删除分支

git branch -d <分支名> # 删除分支, 假如有一个分支不需要合并, 此时删除就需要使用-D参数来强制删除该分支

合并分支

git merge <分支名> # 将某分支合并到当前分支, 快速合并和手动合并(有冲突), 使用Fast forward模式, 下面禁用该模式
git merge --no-ff -m "message" <分支名> # 将某分支合并到当前分支的同时创建一个commit来备注

保存工作现场

git stash # 将当前工作现场保存起来, git status是干净的
git stash list # 查看所有的工作现场
git stash apply # 恢复工作现场, stash内容不删除, 需要使用git stash drop来删除
git stash pop # 恢复工作现场的同时删除stash内容

复制某次提交

git cherry-pick <commit> 将该次提交复制到当前分支上, 可以在多条分支上修复同一个bug

查看远程仓库信息

git remote -v

标签

git tag <tag> <commit> # 给某次commit打上标签, 好记, 若没有commit参数, 就是默认当前分支的最新commit
git tag # 查看所有标签
git tag <tag> -m "message" <commit> # 创建带有说明的标签
git tag -d <tag> # 删除标签
